1
สร้าง Google Sheets
ฐานข้อมูลหลักของระบบ
1.1 สร้าง Spreadsheet ใหม่
  1. เปิด Google Sheets → สร้างใหม่
  2. ตั้งชื่อ: ระบบรวมแถว ป.5 พัน.5
  3. คัดลอก Spreadsheet ID จาก URL:
    https://docs.google.com/spreadsheets/d/[นี่คือ ID]/edit
1.2 ระบบจะสร้าง Sheet อัตโนมัติเมื่อรัน setupSheets()
  • ทะเบียนกำลังพล — LineUserId, ชื่อเต็ม, ยศ, ชื่อ, เบอร์โทร, วันลงทะเบียน, LINE Name, สถานะ
  • รายงานรวมแถว — วันที่, LineUserId, ชื่อ, สถานะ, เหตุผล, เวลา, สถานะเวลา
  • สรุปรายวัน — วันที่, มา, ไม่มา, ไม่รายงาน, รวม, อัตรามา%
  • ประวัติภาพที่ส่ง — บันทึกการส่งภาพ LINE ทุกครั้ง
✅
บันทึก Spreadsheet ID ไว้ก่อน จะใช้ใน Config ขั้นตอนที่ 2
2
ตั้งค่า Google Apps Script
Backend หลักของระบบ
2.1 สร้าง GAS Project
  1. เปิด Google Sheets → เมนู Extensions → Apps Script
  2. ลบโค้ดเดิมออกทั้งหมด
  3. วางเนื้อหา Code_final.gs ลงใน Code.gs
2.2 อัปโหลดไฟล์ HTML
  1. กด + → HTML → ตั้งชื่อ admin_parade
  2. วางเนื้อหา admin_parade_final.html
  3. ทำซ้ำสำหรับ liff_parade และ liff_register
⚠️
ชื่อไฟล์ใน GAS ต้องตรงกัน กับที่ HtmlService.createHtmlOutputFromFile("admin_parade") เรียก — ไม่ต้องใส่ .html
2.3 แก้ไข CONFIG ใน Code.gs
GAS
// แก้ค่าเหล่านี้ก่อน deploy
const CONFIG = {
  SPREADSHEET_ID:      "1BxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxY",
  TELEGRAM_BOT_TOKEN:  "1234567890:AAF...",
  TELEGRAM_CHAT_ID:    "-1001234567890",  // ติดลบถ้าเป็นกลุ่ม
  LINE_CHANNEL_TOKEN:   "eyJhbGciOi...",   // Channel Access Token
  LINE_COMMANDER_ID:    "Uxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
};
2.4 รัน setupSheets() ครั้งแรก
  1. เลือกฟังก์ชัน setupSheets → กด Run ▶
  2. อนุญาต Permission ทั้งหมด (Google Sheets, Drive, UrlFetch)
  3. จะขึ้น Alert "ตั้งค่า Sheets เรียบร้อย ✅"
2.5 Deploy เป็น Web App
  1. กด Deploy → New Deployment
  2. Type: Web App
  3. Execute as: Me
  4. Who has access: Anyone
  5. กด Deploy → คัดลอก Web App URL
⚠️
ทุกครั้งที่แก้ Code.gs ต้อง Deploy ใหม่ (New Deployment หรือ Manage Deployments → Update) ถึงจะมีผล
2.6 ใส่ Web App URL ใน HTML ทุกไฟล์
JS
// แก้ทุกไฟล์ HTML ทั้ง 3 ไฟล์
const GAS_URL = "https://script.google.com/macros/s/AKfy.../exec";
3
ตั้งค่า LINE Developers
LIFF App + Messaging API + Webhook
3.1 สร้าง Provider และ Channel
  1. เข้า developers.line.biz
  2. Create Provider → ตั้งชื่อหน่วย
  3. Create Channel → เลือก LINE Login (สำหรับ LIFF)
  4. สร้างอีก 1 Channel → Messaging API (สำหรับ Bot)
3.2 สร้าง LIFF Apps (2 ตัว)
  1. เข้า LINE Login Channel → แท็บ LIFF → Add
  2. LIFF #1 — ลงทะเบียน:
    Name: ลงทะเบียนกำลังพล, Size: Full, Endpoint: [GAS_URL]?page=register
  3. LIFF #2 — รายงานรวมแถว:
    Name: รายงานรวมแถว, Size: Full, Endpoint: [GAS_URL]?page=parade
  4. คัดลอก LIFF ID ทั้ง 2 ใส่ใน HTML
JS
// liff_register_final.html
await liff.init({ liffId: "1234567890-xxxxxxxx" });

// liff_parade_final.html
await liff.init({ liffId: "1234567890-yyyyyyyy" });
3.3 ตั้งค่า Messaging API Channel
  1. เข้า Messaging API Channel → Basic Settings → คัดลอก Channel Access Token
  2. แท็บ Messaging API → Webhook URL:
    [GAS_URL] → Verify → Use webhook: ON
  3. Auto-reply messages: Disabled
  4. Greeting messages: แก้ตามต้องการ
3.4 หา LINE User ID ผบ.ร้อย
  1. ให้ ผบ.ร้อย ส่งข้อความอะไรก็ได้มาที่ Bot
  2. ดู Logger ใน GAS: Logger.log(ev.source.userId)
  3. นำ User ID ใส่ LINE_COMMANDER_ID ใน Config
💡
หรือใช้ LINE Bot API /v2/profile ก็ได้ถ้ามี User ID อยู่แล้ว
4
สร้าง Telegram Bot
สำหรับแจ้งเตือนเสมียนและกลุ่มหน่วย
4.1 สร้าง Bot ผ่าน @BotFather
  1. เปิด Telegram → ค้นหา @BotFather
  2. พิมพ์ /newbot → ตั้งชื่อ เช่น ป.5 พัน.5 Bot
  3. Username: Pan5Bot (ต้องลงท้าย bot)
  4. คัดลอก Bot Token ใส่ TELEGRAM_BOT_TOKEN
4.2 หา Chat ID กลุ่มหน่วย
  1. เพิ่ม Bot เข้ากลุ่ม Telegram
  2. ส่งข้อความในกลุ่ม
  3. เปิด URL นี้ในเบราว์เซอร์:
    https://api.telegram.org/bot[TOKEN]/getUpdates
  4. หา "chat":{"id": — เลขติดลบคือ Group ID
  5. ใส่ใน TELEGRAM_CHAT_ID
5
Deploy & ทดสอบ
ทดสอบทีละ Feature
  • ✓
    ลงทะเบียนกำลังพล — เปิด LIFF ลงทะเบียน → ตรวจ Sheet ทะเบียนกำลังพล ว่าบันทึกครบ
  • ✓
    รายงาน "มารวมแถว" — เปิด LIFF รายงาน → กด "มารวมแถว" → ตรวจ Sheet รายงานรวมแถว
  • ✓
    รายงาน "ไม่มารวมแถว" — กด "ไม่มา" + เลือกเหตุผล → ตรวจ Telegram ว่าได้รับแจ้ง
  • ✓
    Admin Dashboard — เปิด GAS URL → ตรวจว่า KPI แสดงถูกต้อง
  • ✓
    ส่งภาพ LINE — กด "ส่งภาพ LINE ผบ.ร้อย" → ตรวจ LINE ของ ผบ.ร้อย ว่าได้รับภาพ
  • ✓
    LINE Bot Webhook — พิมพ์ "ยอด" ใน LINE Bot → ต้องได้รับสรุปยอด
  • ✓
    Export CSV — กดปุ่ม CSV ใน Admin → ไฟล์ต้องดาวน์โหลดและเปิดได้ใน Excel
  • ✓
    Telegram Summary — รัน sendDailySummary() manually → ตรวจกลุ่ม Telegram
6
ตั้ง Time-based Triggers
ระบบอัตโนมัติรายวัน

รันฟังก์ชัน setupAllTriggers() ครั้งเดียว — ระบบจะตั้ง trigger ทั้งหมดให้อัตโนมัติ

07:45 น.
reminderNotReported() — ส่ง LINE Push แจ้งกำลังพลที่ยังไม่รายงาน ก่อน cutoff 15 นาที
08:00 น.
ปิดรับรายงาน — รายงานหลังจากนี้จะถูกบันทึกว่า "เกินเวลา"
08:05 น.
sendDailySummary() — ส่งสรุปยอดรวมแถวไปยัง Telegram + บันทึก Sheet สรุปรายวัน
GAS
// รันใน GAS Editor → ตรวจ Triggers ใน Project Settings
setupAllTriggers();  // ตั้ง 07:45 reminder + 08:05 summary
✅
ตรวจ Triggers ได้ที่ GAS → Triggers (นาฬิกา ⏰) ด้านซ้าย — ต้องเห็น 2 trigger

Flow การทำงานรายวัน
ขั้นตอนตั้งแต่เช้าถึงสรุป
ก่อน 07:45 น.
กำลังพลเปิด LINE → กด LIFF รายงานรวมแถว → เลือกมา/ไม่มา → บันทึก Google Sheets
07:45 น. (อัตโนมัติ)
Trigger ส่ง LINE Push แจ้งกำลังพลที่ยังไม่รายงาน พร้อมเบอร์โทร
07:45–08:00 น.
กำลังพลที่ได้รับแจ้งเตือนทำการรายงาน — ยังถูกบันทึกว่า "ทันเวลา"
08:00 น.
ระบบเริ่มบันทึก "เกินเวลา" — LIFF แสดง warning badge สีแดง
08:05 น. (อัตโนมัติ)
Trigger ส่งสรุปยอดรวมแถวไปยัง Telegram กลุ่มหน่วย + บันทึก Sheet สรุปรายวัน
หลัง 08:05 น.
เสมียนเปิด Admin Dashboard → ตรวจยอด → กด "ส่งภาพ LINE ผบ.ร้อย" → ผบ.ร้อย ได้รับภาพสรุปทันที
ตลอดวัน
ผบ.ร้อย พิมพ์ "ยอด" ใน LINE Bot ได้ตลอดเวลา — ได้รับสรุปแบบ real-time
🔧
แก้ปัญหาที่พบบ่อย
❌ LIFF ไม่โหลด / ขึ้น Error
  • ตรวจ LIFF ID ใน HTML ว่าตรงกับ LINE Developers
  • LIFF Endpoint URL ต้องเป็น HTTPS — GAS Web App URL เป็น HTTPS อยู่แล้ว
  • ตรวจ Scope: ต้องเปิด profile และ openid
❌ POST ข้อมูลแล้วไม่บันทึก
  • GAS Web App ต้อง Deploy เป็น "Anyone" — ตรวจ Deployment settings
  • ถ้าแก้ Code.gs แล้วต้อง Deploy ใหม่ เสมอ
  • ตรวจ GAS Logger: Extensions → Apps Script → Executions
❌ Telegram ไม่ได้รับข้อความ
  • ตรวจ Bot Token — ต้องไม่มี space หรือ newline
  • Chat ID กลุ่มต้องมีเครื่องหมาย - นำหน้า
  • Bot ต้องเป็น Admin ในกลุ่ม (หรืออย่างน้อย member)
❌ ส่งภาพ LINE ไม่สำเร็จ
  • ตรวจ LINE_CHANNEL_TOKEN — ต้องเป็น Long-lived token จาก Messaging API
  • ตรวจ LINE_COMMANDER_ID — ต้องเป็น User ID ไม่ใช่ Display Name
  • ไฟล์ใน Google Drive ต้อง public — ตรวจ sharing settings
  • ดู error ใน GAS Logger → sendImageToLine
❌ LINE Bot Webhook ไม่ทำงาน
  • Webhook URL ใน LINE Developers ต้องตรงกับ GAS Web App URL
  • กด Verify ใน LINE Developers — ต้องขึ้น Success
  • Use webhook ต้อง ON
  • GAS ต้อง Deploy ล่าสุด (ไม่ใช่ version เก่า)
